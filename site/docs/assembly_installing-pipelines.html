Installing Pipelines

OpenShift Pipelines can be installed by using the OpenShift Pipelines Operator indexed in the OpenShift OperatorHub. To install the OpenShift Pipelines Operator you need to create a subscription and then create the relevant Custom Resources and Custom Resource Definitions (CRDs).

Installing the Pipelines Operator

Prerequisites

  • An OpenShift 4 cluster is required for installation. Visit try.openshift.com for information on setting up an OpenShift 4 cluster.

Procedure

  1. In the OpenShift Web Console, navigate to Home → Projects. Click Create Project.

  2. In the Create Project dialog box, type pipelines as the name of the project and click Create.

  3. Navigate to Operators → OperatorHub.

  4. Use the Filter by keyword box to search for OpenShift Pipelines Operator in the catalog. Click on the OpenShift Pipelines Operator tile.

  5. The OpenShift Pipelines Operator screen providing a brief description of the operator is displayed. Click Install.

  6. On the Create Operator Subscription screen, retain the default entries, and click Subscribe.

  7. In the Subscription Overview screen, verify the installation of the operator by confirming that the UPGRADE STATUS is 1 installed.

Creating Custom Resources for the Pipelines Operator

Prerequisites

  • Ensure that the OpenShift Pipelines Operator is installed. For more information, see Installing the OpenShift Pipelines Operator.

Procedure

  1. In the OpenShift Pipelines Operator Subscription Overview screen, click the 1 installed link.

  2. In the OpenShift Pipelines Operator Overview tab, click Create New in the Provided APIs section to create an instance of the Openshift Pipelines install CRD. Note that you may need to refresh the screen, in case you do not see the section.

  3. In the Create Install custom resources editor screen, ensure that the namespace displayed is pipelines and click Create.

    The editor is pre-populated with definitions. You can configure the file by manually entering YAML or JSON definitions into the editor.

  4. The pipeline resources are now installed. In the OpenShift Pipelines Install tab, click on example-install to view the Openshift Pipeline resources installed by the operator.

  5. Check on OpenShift Pipelines Operator and resources status by navigating to Workloads → Pods.
